# Optimization Patterns

1. **Schema Compression** — сокращение описаний полей, flatten вложенных структур, использование enum вместо freeform string.
2. **Dynamic Tools** — передавать только нужные тулы per-request, а не все сразу.
3. **Router Pattern** — первый вызов с лёгким роутером который определяет нужную категорию, второй вызов уже с тулами этой категории.
4. **Shadow Tool — тула с минимальной схемой. Невалидный запрос → возвращает схему. Валидный запрос → выполняет.**
5. **Semantic Tool RAG** — перед вызовом LLM эмбеддингом отбираются только релевантные тулы из каталога.
6. **Hierarchical / Meta-tool** — один мета-тул принимает `{category, action, params}`, скрывая от LLM детальные схемы всех подтулов.
7. **CodeAct** — LLM пишет код который оркестрирует несколько вызовов, промежуточные результаты не попадают в контекст.
