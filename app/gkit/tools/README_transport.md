# Genkit Tools Transport

Этот пакет содержит определения инструментов для Genkit AI.

## Принцип работы

Каждый файл в этом пакете соответствует одному унифицированному инструменту Genkit (например, `entities.go` -> инструмент `entities`).

Инструменты в этом слое:
1. Принимают `Input` из пакета `models/`.
2. Валидируют контекст (через `ai.ToolContext`).
3. Вызывают соответствующий Service из пакета `adapters/`.
4. Возвращают результат, который Genkit отправит обратно LLM.

## Реестр (Registry)

Файл `registry.go` отвечает за:
- Хранение ссылок на все необходимые сервисы (Dependency Injection).
- Регистрацию всех инструментов в инстансе Genkit.
